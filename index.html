02
Live Transaction Dashboard
Apache KafkaASP.NET CoreSignalRReactConfluent.Kafka
A real-time transaction streaming platform. Independent Kafka consumers process live events concurrently, pushing a live feed, rolling stats, and anomaly alerts to the UI simultaneously.
Live Demo GitHub
Why I built it · Why these choices
What drove it
I wanted to actually understand Kafka beyond tutorials - so I built something with real concurrency requirements where the architecture would matter.
Why Kafka?
Consumer groups let the feed, stats, and anomaly services each process the same stream independently - no coupling, no shared state.
Why SignalR?
Native .NET real-time push. No polling, persistent WebSocket connection, and the UI updates the moment an event hits.
